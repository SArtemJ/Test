

model.go - файл с описанием используемых структур

database.go - основные функции по работе с БД
- инициализация подключения
- разбор файла конфига myconf.toml

myconf.toml - файла конфига
в котором содержатся
-параметры для подключения к БД
-значение метрики для ошибки
-логин пароль отправителя e-mail

mail.go - файл с функцией отправки e-mail

main.go - файл старта программы
- функция создающая записи метрик и проверки
- как только в одной из метрик нового девайса встречается значение указанное как негативное(myconf.toml)
вызывается функция отправки email

dmclean.sql - дамп БД в котором отражено 10000 пользователей и 10000 устройств.
Таблицы метрик и сообщений пустые

Процесс запуска:
развернуть копию БД на localhost
можно собрать проект: go build
или запустить Test.exe из cmd windows/linux

в процессе работы отразится
-сколько юзеров и девайсов было прочитано из таблиц
-список e-mail на которые отправляются сообщения когда параметр одной из метрик равен значению из файла myconf.toml
